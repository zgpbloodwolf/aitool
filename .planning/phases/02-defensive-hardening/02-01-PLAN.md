    2. Replace it with a deep-copy approach using structuredClone:
       ```typescript
       let messageCopy: typeof msg
       try {
         messageCopy = structuredClone(msg)
       } catch {
         messageCopy = msg
       }
       const tagged = { type: 'io_message', channelId, message: messageCopy }
       ```

    Why structuredClone instead of JSON.parse(JSON.stringify()):
    - structuredClone is a V8 built-in, available in Electron's Node.js without imports
    - Handles Date, RegExp, Map, Set, ArrayBuffer, typed arrays correctly
    - JSON.parse/stringify loses Date objects (converts to string), undefined values, etc.
    - Performance: structuredClone is faster than JSON round-trip for typical message sizes

    Why try/catch:
    - structuredClone throws for non-cloneable values (functions, symbols, WeakMap, etc.)
    - If claude.exe ever emits a message containing a function proxy or similar, the app should not crash
    - Fallback to original msg ensures no message is lost

    The placement matters: ALL reads from `msg` (plan text capture, tool permission tracking, token tracking) happen BEFORE this line, operating on the original object. Only the forward-to-webview path gets the copy.
