    **a) D-13: 智能标签名**

    目标：当收到 `system` 消息且 subtype 为 `'init'` 时，尝试从消息中提取摘要文本，更新当前 tab 的 label。

    1. 在 `handleSystemMessage` 的 `case 'init'` 分支中：
       - 检查 `msg` 中是否有 `summary` 字段（类型 `string`）。如果存在且非空，截取前 20 个字符作为标签名
       - 检查 `msg` 中是否有 `session_id` 字段（类型 `string`）。如果存在，尝试调用 `window.api.claudeListSessions()` 获取会话列表，查找匹配 `session_id` 的会话的 `summary` 字段，截取前 20 字符
       - 如果两种方式都无法获取，保持 tab 的当前 label（即 "对话 N" 格式，由 `addNewTab()` 设置）
       - 使用当前 tabId 查找：遍历 `channelToTab` 找到 `_channelId` 对应的 tabId，然后在 `tabs.value` 中更新该 tab 的 `label`

    2. 实现示例代码（在 `case 'init'` 分支内，`showStatus(...)` 之后）：
       ```typescript
       // 提取智能标签名
       let newLabel: string | null = null
       if (typeof msg.summary === 'string' && msg.summary.trim()) {
         newLabel = msg.summary.trim().slice(0, 20)
       } else if (typeof msg.session_id === 'string') {
         try {
           const sessions = await window.api.claudeListSessions()
           const match = sessions.find(s => s.id === msg.session_id)
           if (match?.summary) {
             newLabel = match.summary.trim().slice(0, 20)
           }
         } catch { /* 获取失败不影响流程 */ }
       }
       if (newLabel) {
         const tabId = channelToTab.get(_channelId)
         if (tabId) {
           const tab = tabs.value.find(t => t.id === tabId)
           if (tab) tab.label = newLabel
         }
       }
       ```
       - 注意：`handleSystemMessage` 当前是普通函数，如果使用 `await` 需要改为 `async`。如果不想改函数签名，可以将 `claudeListSessions()` 调用放在 `.then()` 中

    **b) D-14: 标签状态指示器**

    1. 在 `<script setup>` 中添加响应式状态 Map：
       ```typescript
       const tabStatuses = ref<Map<string, 'running' | 'waiting' | 'idle'>>(new Map())
       ```

    2. 在 `handleSystemMessage` 中更新状态：
       - 在 `case 'init'` 分支：查找 `_channelId` 对应的 tabId，设置 `tabStatuses.value.set(tabId, 'running')`
       - 在 `case 'hook_started'` 分支：同上，设置 `'running'`
       - 在 `case 'result'` 分支（在 `clearStatus()` 之后）：设置 `'idle'`
       - 在其他非 result 的 system 消息（`default` 分支）：设置 `'running'`（如果 tabId 可找到）

    3. 在 `forwardToWebview` 中检测权限请求消息：
       - 当 `m?.message?.type === 'request'`（权限请求）时，获取 tabId 并设置 `tabStatuses.value.set(tabId, 'waiting')`
       - 在 `m?.message?.type === 'response'`（权限响应，如果有的话）或 result 消息时，恢复为 `'idle'`

    4. 在模板的 `.tab-item` 中添加状态圆点：
       - 在 `<span class="tab-label">` 之前添加：
         ```html
         <span class="tab-status-dot" :class="tabStatuses.get(tab.id) || 'idle'" />
         ```

    5. 添加 CSS 样式（在 `<style scoped>` 中）：
       ```css
       .tab-status-dot {

         border-radius: 50%;
         flex-shrink: 0;
       }
       .tab-status-dot.running {
         background: var(--success);
         animation: pulse 1.5s ease-in-out infinite;
       }
       .tab-status-dot.waiting {
         background: var(--warning);
         animation: pulse 1.5s ease-in-out infinite;
       }
       .tab-status-dot.idle {
         background: var(--text-muted);
       }
       ```
       - 注意：`pulse` 动画的 `@keyframes` 已在文件中存在（用于 status-dot），可复用
